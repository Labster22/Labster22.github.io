2目录结构介绍

原生Hexo目录结构

Code
  • node_modules 是npm安装插件的位置
  • public 是执行hexo g命令后，输出的静态网页内容目录
  • scaffolds layout模板文件目录，之后生成指定文件时会依据不同模板，例如
Code

​ 这是一个文章模板，title用于写题目，date为时间，tags可以添加不同的标签，categoties是文章所属的目录，cover是文章表面的图片……

  • scripts 存放扩展脚本

  • source 文章源码目录，存放md文件

  • themes 主题目录，可以根据个人喜好在网上下载主题后添加至此

  • _config.yml 全局配置文件，主要配置都在这

  • package.json 模块的描述文件，与node.js和npm有关

  • package-lock.json 基本同上
